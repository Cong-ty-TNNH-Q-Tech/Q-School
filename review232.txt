PR #232 triển khai tính năng AI Chat cốt lõi có rất nhiều điểm cộng:
- Triển khai SSE (Server-Sent Events) với `StreamingResponse` thay vì WebSocket rất chuẩn xác theo Architecture Guidelines.
- Sử dụng `anyio.CancelScope(shield=True)` để đảm bảo lưu AI message vào DB ngay cả khi client ngắt kết nối là một kỹ thuật xuất sắc!

Tuy nhiên, mình xin **Request Changes** vì PR đang vi phạm 3 quy tắc quan trọng của dự án:

### 🚨 Bắt buộc phải sửa

1. **Vi phạm Hexagonal Architecture (Rò rỉ Business Logic ra Router)**
   - Trong file `ai_chat.py` (ở các endpoint `get_chat_messages` và `send_chat_message`), bạn đang khởi tạo trực tiếp `chat_repo = ChatSQLAlchemyRepository(db)` và kiểm tra `session.user_id != user.id`.
   - Lời giải thích `TODO: ... logic đó có thể được xử lý ở router layer` trong UseCase là **SAI** nguyên tắc. Router chỉ làm nhiệm vụ nhận/trả HTTP request. Toàn bộ logic kiểm tra quyền sở hữu (ownership) và fetch session phải nằm gọn trong `ChatUseCase`.
   - **Cách fix**: Đưa đoạn logic lấy session và check `user_id` vào trong UseCase, sau đó ném ra các Domain Exception (vd: `SessionNotFoundError`, `PermissionDeniedError`) rồi map sang HTTP 404/403 ở tầng Router.

2. **Sai cấu trúc Response chuẩn (API Envelope)**
   - Các API như `create_chat_session`, `get_chat_sessions`, `get_chat_messages` đang trả về thẳng data model (VD: `return ChatSessionListResponse(...)`).
   - Theo chuẩn trong `AGENTS.md` và `openapi.yaml`, tất cả JSON response ngoại trừ SSE stream đều phải bọc trong API Envelope.
   - **Cách fix**: Sử dụng `ApiResponse` cho các endpoint (ngoại trừ endpoint stream). Ví dụ: `response_model=ApiResponse[ChatSessionListResponse]` và return `ApiResponse(data=...)`.

3. **Lỗi tiềm ẩn trong Cursor Pagination của `get_messages`**
   - Ở `chat_repository.py`, hàm `get_messages` gọi `cursor_paginate` nhưng set cứng `cursor_id=None`.
   - **Cách fix**: Cần truyền `cursor_id` từ param xuống để làm tie-breaker. Tin nhắn sinh ra rất nhanh (của User và AI liền nhau) có thể trùng `created_at` đến tận millisecond, nếu không có ID làm tie-breaker thì phân trang sẽ bị lặp/sót dữ liệu.

Vui lòng cập nhật các điểm trên để đảm bảo dự án chạy ổn định và chuẩn thiết kế nhé. Cảm ơn bạn!
