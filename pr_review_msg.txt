### 1. Vi phạm Nguyên tắc Dependency Inversion và Hexagonal (Vòng 3 & Vòng 4)
- **File:** `backend/app/entrypoints/api_v1/billing.py`
- **Lỗi:** Router đang tự khởi tạo `BillingRepository(db)` và `CreateCheckoutSessionUseCase(repo)` trực tiếp bằng tay. Đồng thời gọi luôn `await db.commit()` ở tầng Router.
- **Phân tích:** Điều này vi phạm nghiêm trọng chữ **D (Dependency Inversion)** trong SOLID và vi phạm trực tiếp quy tắc trong `AGENTS.md` (BẮT BUỘC dùng cơ chế DI của FastAPI để inject Repository vào Use Case). Router (Driving Adapter) KHÔNG ĐƯỢC PHÉP chứa logic xử lý transaction hay khởi tạo class cấp thấp.
- **💡 Suggested Code:**
Cần tạo các hàm Dependency Injection (ví dụ `get_checkout_use_case`) và ủy quyền việc `commit` cho tầng Use Case hoặc Repository (Unit Of Work).

```python
# Xóa bỏ việc khởi tạo thủ công và db.commit() trong router
@router.post("/checkout", response_model=CheckoutResponse)
async def create_checkout_session(
    request: CheckoutRequest,
    user_id: uuid.UUID = Depends(get_current_user_id),
    use_case: CreateCheckoutSessionUseCase = Depends(get_checkout_use_case)
):
    # Use Case sẽ tự lo việc xử lý logic và commit dữ liệu
    result = await use_case.execute(
        user_id=user_id, 
        plan_id=request.plan_id, 
        provider=request.provider
    )
    
    return CheckoutResponse(
        status="success",
        data=CheckoutResponseData(
            checkout_url=result["checkout_url"],
            transaction_id=result["transaction_id"]
        )
    )
```

### 2. Nguy cơ lỗi Migration Database (Vòng 8 - Backward Compatibility)
- **File:** `backend/app/domain/models/billing.py`
- **Lỗi:** Thay đổi `CheckConstraint` của trường `provider`.
- **Phân tích:** Việc đổi từ `('stripe', 'vnpay', 'momo')` thành độc nhất `('sepay')` là một lỗi **phá vỡ tương thích ngược**. Nếu trên Database Production đã tồn tại các giao dịch cũ có provider là `stripe` hoặc `vnpay`, khi chạy lệnh `alembic upgrade`, hệ thống sẽ ném lỗi `IntegrityError` (Data Constraint Violation) và làm sập luồng deploy. Theo nguyên tắc Open/Closed, chúng ta chỉ nên MỞ RỘNG enum.
- **💡 Suggested Code:**
```diff
- "provider IN ('sepay')"
+ "provider IN ('stripe', 'vnpay', 'momo', 'sepay')"
```
